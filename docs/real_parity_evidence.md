# Real Parity Evidence Gate

## Purpose

The real parity evidence gate compares a TradingView/Pine diagnostic export against a Python mirror output for the exact same strategy, symbol, timeframe and bar interval.

This is a P0 release gate. The project remains `HOLD` until real exports show zero critical mismatches.

## Required input files

The runner expects two CSV files:

1. Pine diagnostic export from TradingView.
2. Python mirror output generated by this repository.

Both files must contain a `timestamp` column and the required signal columns:

```text
timestamp
EXP_longSignal
EXP_shortSignal
```

Recommended columns:

```text
EXP_state
EXP_score
```

Additional overlapping diagnostic columns are compared by the existing parity checker.

## Run

```bash
python scripts/run_real_parity_evidence.py \
  --pine-export data/pine_exports/amlrx/BTCUSDT/5m/amlrx_BTCUSDT_5m_export.csv \
  --python-output data/processed/amlrx_BTCUSDT_5m_python.csv \
  --strategy amlrx \
  --symbol BTCUSDT \
  --timeframe 5m \
  --json-out reports/parity/amlrx/BTCUSDT/5m/parity_evidence.json \
  --md-out reports/parity/amlrx/BTCUSDT/5m/parity_evidence.md
```

## GO condition

The report only passes when all conditions hold:

```text
critical_mismatches = 0
signal_bar_match_rate = 1.0
direction_match_rate = 1.0
```

## HOLD condition

The report returns `HOLD` if any of these occur:

- Pine export file is missing.
- Python output file is missing.
- Required signal columns are absent.
- Signal bars do not match.
- Direction does not match.
- Index/timestamp overlap is missing.
- Any critical mismatch exists.

## Release stance

This gate is intentionally strict. Synthetic data can verify code paths, but real TradingView exports are required before the project can move from `HOLD` to `GO`.
